# ARD-0003 — Frontend Framework and PWA Strategy

**Status:** Accepted
**Date:** 2026-06-24
**Owner:** Engineering
**Type:** Frontend / PWA
**Required Before:** Milestone 1
**Source Alignment:** PRD, database design/schema, architecture, API contracts, QA plan, RTM, user stories, permission matrix, UX map, tech stack, roadmap, architecture records

## Context

GarageOS must be a mobile-first PWA. Offline support is limited to the app shell and read-only recent-record cache.

## Decision

Use **Next.js, React, and TypeScript** for the web app.

Implement PWA support with:

- Web app manifest
- Service worker
- App-shell caching
- Read-only recent-record cache

Offline mode must:

- Show an offline indicator.
- Allow app shell access and recently viewed records in read-only mode.
- Block all writes, approvals, uploads, payments, inventory actions, and sync behavior.
- Clear user-scoped cache on logout or session invalidation.

## Consequences

- Supports mobile-first UX and installability.
- Avoids unsupported offline-write complexity.
- Requires careful service worker cache invalidation and authenticated cache controls.

## Validation

- Playwright covers PWA shell, offline indicator, blocked offline writes, and cache clearing on logout.
- QA validates core workflows at 360px minimum width.

## Risks

- Cache mistakes may expose stale or unauthorized data.
- Browser/device PWA behavior may vary.
- Accidental offline writes would violate product exclusions.

## Follow-ups

- Define the service worker cache strategy.
- Add Playwright mobile/offline scenarios.
- Document logout/session cache cleanup behavior.
