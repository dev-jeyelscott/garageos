# ARD-0011 — Identifier Strategy

**Status:** Accepted
**Date:** 2026-06-24
**Decision Owner:** Engineering
**Decision Type:** Database / API
**Required Before:** Milestone 1
**Source Alignment:** Core PRD, database, architecture, API, QA, RTM, roadmap, and ARD docs

## Context

GarageOS requires UUID primary IDs, with UUIDv7 preferred. Tenant-scoped document numbers must be separate from IDs and never reused.

## Decision

Use **UUIDv7 in PostgreSQL `uuid` columns** for primary entity IDs.

Rules:

- Generate authoritative IDs server-side.
- Expose public API IDs as UUID strings.
- Do not use ULID/text IDs for primary relational IDs in the initial build.
- Generate tenant-scoped business document numbers separately and protect them with unique constraints plus transaction locks.

## Consequences

UUIDv7 improves index locality and keeps UUID compatibility, but requires a maintained pinned library and may reveal rough creation order.

## Validation

- Unit tests verify valid UUID generation.
- Schema tests verify UUID columns and tenant-scoped document-number uniqueness.
- Concurrency tests prove document numbers are never duplicated or reused.

## Follow-ups

- Select and pin UUIDv7 library.
- Define tenant-scoped document-number allocation.
- Add document-number concurrency tests.
