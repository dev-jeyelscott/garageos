# ARD-0023 — Environments, Deployment Units, and CI/CD

**Status:** Accepted
**Date:** 2026-06-24
**Decision Owner:** Engineering
**Decision Type:** DevOps / Deployment
**Required Before:** Milestone 1
**Source Alignment:** Core PRD, database, architecture, API, QA, RTM, user stories, permissions, UX, tech stack, roadmap, and ARD docs

## Context

GarageOS requires local, development, staging, and production environments; containerized PWA/API/worker/scheduler services; CI checks; secrets handling; migration checks; and production-readiness evidence.

## Decision

Use Dockerized deployment units:

| Unit             | Responsibility                                     |
| ---------------- | -------------------------------------------------- |
| PWA static app   | Client shell and assets.                           |
| API service      | REST API, commands/queries, auth, signed URLs.     |
| Worker service   | Jobs, exports, reminders, lifecycle tasks, outbox. |
| Scheduler        | Periodic job enqueueing.                           |
| PostgreSQL       | Transactional source of truth.                     |
| Object storage   | Tenant files and export packages.                  |
| Monitoring stack | Logs, metrics, traces, errors, alerts.             |

Required environments:

- `local`
- `development`
- `staging`
- `production`

CI baseline:

- Lint
- Typecheck
- Unit tests
- Migration validation
- API contract tests
- Practical integration tests
- Dependency/security scanning

## Consequences

This supports predictable releases and staging parity. Worker/scheduler separation protects API capacity but requires disciplined environment config, secrets handling, and scaling.

## Validation

- New developers can run locally from documented steps.
- CI blocks failed lint, type, unit, migration, and contract checks.
- Staging validates migrations, workers, storage sandbox, provider sandboxes, and E2E tests before production.

## Risks

- Environment drift may hide deployment defects.
- Poor config patterns can leak or mishandle secrets.
- Workers/schedulers may be mis-scaled relative to API load.

## Follow-ups

- Document environment variables for all environments.
- Add deployment runbook and rollback checklist.
- Configure staging parity for workers, storage, and provider sandboxes.
